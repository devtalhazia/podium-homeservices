## Architecture

This is a marketing landing page for Podium's "Larry AI" product. It has a dual rendering architecture — most of the page is injected Framer HTML, but sections are being progressively replaced with React components.

### Framer-HTML path (current `Home` page)

`src/pages/Home.tsx` builds a full-page HTML string by concatenating section strings in order:

```
shellOpen + heroHTML + solutionHTML + integrationsHTML + testimonialsHTML
         + communicationsHTML + marketingHTML + ctaHTML + faqHTML + shellClose
```

Each section string lives in `src/data/sections/<name>.ts` — auto-generated by `scripts/extract-sections.mjs` from the monolithic `src/data/homePageHTML.ts` source of truth.

The concatenated string is injected via `dangerouslySetInnerHTML`. Because that does not execute `<script>` tags, `framerScripts.ts` re-creates them via `document.createElement('script')`, appending Framer data scripts (`id="__framer__*"`) inside the container and all other scripts to `<head>`.

### React-component path (in progress)

`src/sections/HeroSection.tsx` is a full React + framer-motion rebuild of the hero section. The Framer-path `hero.ts` file currently exports only `<div id="hero-react-mount"></div>`, which is the intended React mount point. `/test/hero` route lets you preview this component in isolation.

Test routes: `/test/faq` and `/test/hero` — standalone previews for individual sections under development.

### CSS / design tokens

- `src/styles/framer.css` — Framer-generated CSS (large; don't hand-edit)
- `src/styles/globals.css` — global overrides
- `src/styles/hero.css` — styles for the React hero rebuild

Tailwind is configured with Framer's exact design tokens as named colors (`nav`, `ink`, `cream`, `crimson`, etc.) and font families (`graphik`, `grenette`, `editors-note`, `fragment-mono`). Framer breakpoints: `tablet` = 768 px, `desktop` = 1200 px.

### Image CDN

All images reference `https://framerusercontent.com/images/<hash>` with Framer's query-string resizing (`?width=N`, `?scale-down-to=N`). The constant `BASE` in `HeroSection.tsx` holds this root URL.

### Re-extracting sections

If `homePageHTML.ts` is updated, re-run:

```bash
node scripts/extract-sections.mjs
```

The script validates that `shellOpen + sections + shellClose` reconstructs the original HTML exactly before writing any files.
